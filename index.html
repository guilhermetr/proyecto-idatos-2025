Notas metodológicas

  • Los datos se consultan a través de una función serverless en Vercel (/api/integrate) que descarga y normaliza las fuentes públicas.
  • No se mantiene una base de datos persistente: la función actúa como mediador y devuelve datos integrados bajo demanda.
  • Si alguna fuente falla, cambia estructura o presenta problemas de acceso, esa parte no aparecerá en el tablero hasta que la fuente sea compatible.

Trazabilidad de las fuentes

  • Clima: INUMET (temperatura, precipitación).
  • Producción / Rendimientos / Precios consumidor: INIA / INE / MEF (datasets públicos).
  • No se almacena información de forma persistente; la función serverless integra y devuelve datos frescos en cada consulta.

Este tablero descarga y normaliza datos públicos, cruza clima con producción y precios por día y región, y los muestra de forma filtrable.
